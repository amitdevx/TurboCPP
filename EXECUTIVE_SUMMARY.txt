  Problem: DOS emulation uses 80-100% CPU cycles continuously
  Impact:  Excessive heat, battery drain on laptops
  Cause:   Inherent to 16-bit DOS on 64-bit systems
  Status:  Fixed with cycle limiting

ISSUE 2: No CPU Cycle Limiting (SEVERITY: MEDIUM)
  Problem: Runs at maximum capacity without restrictions
  Impact:  Unnecessary CPU load during idle time
  Cause:   Missing configuration parameters
  Status:  Easy to fix in start.sh

ISSUE 3: Missing Memory Constraints (SEVERITY: MEDIUM)
  Problem: No memory allocation caps
  Impact:  Potential memory bloat on extended use
  Cause:   Missing -memsize parameter
  Status:  Fixable with single option

ISSUE 4: No Idle Pause Feature (SEVERITY: MEDIUM)
  Problem: Idle loops consume CPU continuously
  Impact:  Battery drain, unnecessary heat generation
  Cause:   No pause-on-idle implementation
  Status:  Can be addressed with scripting

ISSUE 5: Obsolete Swap File (SEVERITY: LOW)
  Problem: TC0000.SWP (256KB) file leftover
  Impact:  File system clutter
  Cause:   Not cleaned up after project migration
  Status:  Simple cleanup

12. TROUBLESHOOTING GUIDE
================================================================================

PROBLEM: Programs run too slowly after optimization
  Cause:   Cycles set too low for your needs
  Fix:     Increase -cycles value:
           - Low speed (slow): cycles=1000
           - Medium speed: cycles=2000
           - High speed: cycles=5000
  Test:    Try Ctrl+F5 and observe execution speed

PROBLEM: CPU usage still above 70%
  Cause:   Cycles limit not enough for system
  Fix:     Lower cycles to 1500
           Or add -nosound to disable audio
  Test:    Monitor with top and adjust

PROBLEM: Graphics/games stutter or lag
  Cause:   Frameskip disabled in configuration
  Fix:     Add to .dosboxrc:
           frameskip=1 or frameskip=2
  Test:    Retry problematic application

PROBLEM: Sound not working or distorted
  Cause:   Audio disabled or misconfigured
  Fix:     Ensure .dosboxrc has:
           nosound=false
           rate=22050
  Test:    Check if sound comes back

PROBLEM: Still overheating on laptop
  Cause:   Insufficient throttling with current profile
  Fix:     Switch to Laptop Profile:
           dosbox -cpu auto -cycles 1000 -nosound
  Test:    Monitor temperature for improvement

PROBLEM: TC0000.SWP file still present after cleanup
  Cause:   File location or permissions issue
  Fix:     Run: find . -name "TC0000.SWP" -delete
  Or:      rm -f /path/to/TC0000.SWP

PROBLEM: Changes not persisting after restart
  Cause:   Configuration not saved properly
  Fix:     Verify .dosboxrc exists in TC directory
           Verify start.sh modifications saved
           Check file permissions: ls -la
